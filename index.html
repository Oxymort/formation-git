Pourquoi versionner ?

  • Backup / Archive
  • Historique
  • Annuler des changements
  • Comparer
  • Travailler en équipe
  • Trouver le coupable :D

Deux types de système

Centralisé

  • Subversion
  • CVS
  • Clearcase
  • Perforce
  • TFS
  • Requiert une connexion au serveur central pour la plupart des opérations

Décentralisé

  • Mercurial (Hg)
  • Git
  • Bazaar
  • La plupart des opérations se font localement
  • Un server central n'est pas nécessaire
  • Le code est présent sur toutes les machines

Qu'est ce que Git ?

  • Système distribué
  • Scale facilement
  • Open source
  • Un server central n'est pas nécessaire
  • Très rapide
  • Grande communauté
  • Système de versionning le plus populaire

Historique

  • Première version en 2005
  • Linus Thorvalds
  • Initialement développé pour le projet Linux
  • Origine du nom :)

Concepts clés

  • Dépot = Repository
  • Contient fichiers, historique et config Git
  • 3 états locaux (repertoire de travail, staging, commit)
  • Dépot distant (GitHub) = source de vérité
  • Branche Master

Ajout d'un fichier

                        
$ echo "Test Git" >> start.txt
$ ls
$ README.md start.txt
$ cat start.txt
$ Test Git
$ git status
$ git add start.txt
$ git status
$ git commit -m "Adding start.txt"
$ git status
$ git push origin master
